Project Overview:
TaskMaster is a productivity and task management mobile app designed to help users organize assignments, tasks, focus sessions, and daily progress. The goal of this app is to go beyond a basic to-do list by including task categories, priorities, difficulty levels, a focus timer, progress tracking, completed task history, calendar planning, and settings.

This Phase 2 prototype focuses on the user interface, navigation flow, and overall user experience. The screens are designed to show how the app will look and how users will move through the app. Some features use mock data or prototype behavior because the final database logic, reminders, advanced statistics, and full functionality will be completed in Phase 3.

Screens and Purpose:

1. Home / Dashboard Screen:
The Home screen is the main dashboard of the app. It displays the app title, daily progress, task list, checkboxes, and navigation options. This screen helps users quickly see what tasks they need to complete and access the main features of the app.

2. Add Task Screen:
The Add Task screen allows the user to create a new task. The screen includes input fields and options such as task title, description, category, priority, difficulty, due date, recurring option, reminder option, and a save button. For this phase, the screen is mainly focused on showing the complete form layout and user flow.

3. Task Details Screen:
The Task Details screen shows more information about a selected task. It displays the task title, description, priority, difficulty, category, due date, and task progress. It also includes buttons for editing the task, deleting the task, and starting a focus timer.

4. Focus Timer Screen:
The Focus Timer screen is a Pomodoro-style timer that helps users focus on one task at a time. It includes a timer display, task selection, focus/break mode options, start/reset controls, session progress, and motivational text. This feature makes the app more useful than a basic task list because it helps users work on tasks, not just list them.

5. Completed Tasks / History Screen:
The Completed Tasks screen shows tasks that have already been finished. This gives users a basic history of their completed work and helps them track productivity over time.

6. Stats / Progress Screen:
The Stats screen displays prototype productivity statistics such as completed tasks, focus sessions, streaks, productivity score, progress bars, and badges. Some information on this screen uses mock data for the Phase 2 prototype.

7. Calendar Screen:
The Calendar screen gives users a calendar-style view of upcoming tasks and due dates. It helps users plan tasks by date instead of only viewing them in a list format. This screen includes mock upcoming tasks and due date examples.

8. Settings Screen:
The Settings screen includes prototype options for notifications, dark mode/theme, daily focus goal, reminder time, and app information. These settings are visually included in the prototype, but some of the actual setting behavior will be completed in the final phase.

Navigation:
The prototype includes basic navigation between the main screens. Users can move from the dashboard to the add task screen, task details screen, focus timer, completed tasks, stats, calendar, and settings. Back navigation is also included where needed.

Design Choices:
The app uses a clean productivity-style design with light backgrounds, card-based layouts, consistent spacing, readable typography, and color-coded UI elements. The design focuses on making tasks easy to read and making the app feel organized and professional.

Resources Used:
The project uses Android XML layout files, colors.xml, strings.xml, themes.xml, drawable resources, and Kotlin activity files. The UI was designed using Android Studio with layouts such as LinearLayout, ConstraintLayout, ScrollView/NestedScrollView, RecyclerView, and Material-style components where appropriate.

Incomplete Features:
This Phase 2 prototype does not fully implement every final feature. Some buttons and screens are included as visual prototype elements. Full database polish, real reminder notifications, advanced statistics, calendar syncing, complete edit/update behavior, and final settings functionality will be completed in Phase 3.

Known Notes:
Some screens may use mock data to demonstrate the intended UI and user experience. This is intentional for Phase 2 because the focus is on UI/UX design and navigation rather than full backend functionality.
